Patients & Records Module (com.cureone.patients)
-----------------------------------------------
Purpose
-------
This module manages patient data and medical records: registration, updates, retrieval of patient history, and linking records to appointments or billing. It is the authoritative source for patient identity and medical history.

Sub-packages & Responsibilities
-------------------------------
1) model
   - Core patient entities and medical records.
   - Classes: Patient, MedicalRecord.
   - Typical fields:
     - Patient: id, name, age, contact
     - MedicalRecord: recordId, patientId, description, date

2) repository
   - Data access for patients and records.
   - Class: PatientRepository (interface).
   - Typical methods: save(patient), findAll(), findById(id), update(patient), delete(id)
   - MedicalRecord storage may be part of PatientRepository or a separate MedicalRecordRepository if desired.

3) service
   - Business logic for patient lifecycle:
     - registerPatient, updatePatient, fetchPatient, listPatients
     - addMedicalRecord, listMedicalRecords
   - Class: PatientService
   - Typical methods:
     - registerPatient(Patient)
     - getPatientById(id)
     - listPatients()
     - updatePatient(Patient)
     - addMedicalRecord(MedicalRecord)
     - listMedicalRecords(patientId)

4) controller
   - User-facing operations for patient workflows.
   - Class: PatientController
   - Typical methods:
     - register(Patient), listPatients(), updatePatient(Patient)

Classes & Key Members (summary)
-------------------------------
- Patient
  - Fields: int id, String name, int age, String contact
  - Purpose: core identity and basic contact information.

- MedicalRecord
  - Fields: int recordId, int patientId, String description, String date
  - Purpose: capture medical notes, diagnoses, treatments tied to a patient.

- PatientRepository (interface)
  - Methods: save, findAll, findById, update, delete
  - Purpose: store/retrieve patient objects and optionally records.

- PatientService
  - Methods: registerPatient, getPatientById, listPatients, updatePatient, addMedicalRecord
  - Purpose: validation, linking records to patients, and data retrieval for other modules.

- PatientController
  - Methods: register, listPatients, updatePatient
  - Purpose: provide simple UI/commands to manage patients.

Notes & Implementation Tips
---------------------------
- Avoid exposing repository implementation details to other modules; expose patient operations via PatientService.
- MedicalRecord association uses patientId to link records; services should enforce that patient exists before adding records.
- Consider a small test runner (main method) for PatientService so reviewers can quickly validate registration and record creation.
